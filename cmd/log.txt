Чтение журнала.
Основная команда для просмотра:
# journalctl
Она выведет все записи из всех журналов, включая ошибки и предупреждения, начиная с того момента,
когда система начала загружаться. Старые записи событий будут наверху, более новые — внизу,
также journalctl позволяет просмотреть логи с временем UTC:
# journalctl --utc
Если мы хотим просмотреть только ошибки, игнорируя другие сообщения, введем команду с указанием
кода важности:
# journalctl -p 0
Для уровней важности, приняты следующие обозначения:
0: emergency (неработоспособность системы)
1: alerts (предупреждения, требующие немедленного вмешательства)
2: critical (критическое состояние)
3: errors (ошибки)
4: warning (предупреждения)
5: notice (уведомления)
6: info (информационные сообщения)
7: debug (отладочные сообщения)
# journalctl -p 1
# journalctl -p 2
# journalctl -p 3

1 удаление лишних логов.
vacuum-size
Этот способ позволяет уменьшить размер директории до указанного размера. Например, так:
sudo journalctl --vacuum-size=50M
vacuum-time
В этом случае мы оставим логи, которые были созданы за последние N дней.
sudo journalctl --vacuum-time=7d
Теперь все файлы старше 7 дней удалены.
vacuum-files
Этот вариант позволяет оставить последний N созданных файлов.
sudo journalctl --vacuum-files=10

2 Ограничение логов
В /etc/systemd/journal.conf настраивается максимальный размер логов. Раскомментировать и задать
sudo gedit /etc/systemd/journald.conf

SystemMaxUse=100M
SystemMaxFileSize=50M
SystemMaxFiles=4
SystemMaxUse=100M предназначено для использования с logrotate
Перезапускаем систему журналирования.
sudo systemctl restart systemd-journald

3 Ограничение количества файлов логов, ротируемых logrotate.
Logrotate — это утилита, которая управляет чередованием и сжатием файлов журналов. Что поможет
уберечь владельцев серверов от остановки сервисов, и недоступности сайтов.
Logrotate каждый день совершает ротацию почти всех лог-файлов, находящихся в папке /var/log.

Конфиг глобальной настройки:
sudo gedit /etc/logrotate.conf
Каталог конфигурации logrotate:
sudo gedit /etc/logrotate.d/rsyslog
дефолтное значение параметра rotate равно 4, можно изменить на другое значение, например 1 и
хранить лишь 1 файл
